User Onboarding Flow
First-time device registration through admin approval to active session.
1
App Launch System
  • Animated splash screen plays for 3 seconds
  • First-time users see a 3-page animated walkthrough (skippable)
  • User selects their role: Admin, Employee, or Supervisor
2
Device Registration Employee / Supervisor
  • User enters their device name on the registration screen
  • System generates an Account Request with device ID and details
  • Request is queued for Administrator review
  • User awaits admin approval before proceeding to login
3
Admin Review Admin
  • Administrator sees a notification badge on dashboard for pending requests
  • Reviews device name, registration details, and user information
  • Approves or rejects the device registration
✅ Approved
  • User proceeds to role-specific login screen
  • Enters Username + Password
  • JWT token returned and stored securely
  • Auto-login active until logout or expiry
❌ Rejected
  • User is notified of rejection
  • Registration attempt blocked
  • User must contact administrator
4
Active Session System
  • JWT Bearer token attached to every API request
  • Session persists until user logs out or token expires
  • On logout: secure storage cleared, returns to role selection
Attendance Check-In / Check-Out Flow
GPS geofencing + mandatory photo verification for tamper-resistant attendance records.
1
Initiate Check-In Employee
  • Employee opens the Check-In screen
  • Selects location type: Office (branch), Customer location, or Site
  • App fetches coordinates for the selected location from server
2
GPS Location Validation System
  • Map displays with geofence polygon boundary highlighted
  • App acquires employee's current GPS coordinates
  • System compares employee location against the polygon boundary
📍 Inside Geofence
  • Camera opens automatically
  • Employee captures mandatory photo
  • Check-in submitted: employee ID, photo (base64), GPS, timestamp
  • Background location tracking starts
🚫 Outside Geofence
  • Error message displayed
  • Check-in blocked — no record created
  • Employee must move to valid location and retry
3
During Work — Background Tracking System
  • Background location service periodically records employee GPS coordinates
  • Supervisor can view team location and attendance status in real time
  • Late comers and early leavers are automatically flagged
4
Check-Out Employee
  • Employee navigates to Check-Out screen at end of shift
  • GPS location re-validated against geofence
  • Second mandatory photo captured for exit verification
  • Check-out submitted; background tracking stops
  • Total work duration calculated and attendance record finalized
5
Supervisor Review & Export Supervisor
  • Reviews team attendance dashboard with check-in times and durations
  • Views late comers list (arrived after shift start)
  • Views early leavers list (left before shift end)
  • Can attend on behalf of employees (if admin-authorized)
  • Exports attendance data to Excel for HR processing
Leave Management Workflow
Complete lifecycle from employee submission through supervisor decision.
1
Submit Leave Request Employee Pending
  • Fills in: Start Date, End Date, Reason (max 200 chars), optional Remark
  • Request created with Pending status
  • Visible in My Leave Requests under the Leave tab
  • Employee can delete the request while it remains Pending
2
Supervisor Notified System
  • Push notification sent to supervisor (when Firebase enabled)
  • Request appears in Supervisor leave approval queue
3
Supervisor Reviews Request Supervisor
  • Reviews leave dates, reason, and employee history
  • Checks team schedule and availability
  • Makes approval decision within target of < 24 hours
✅ Approved Approved
  • Status updated to Approved (green)
  • Employee notified via push notification
  • Leave appears in Leave Planner calendar
  • Added to Leave Schedule timeline
❌ Rejected Rejected
  • Status updated to Rejected (red)
  • Employee notified of rejection
  • Employee can submit a new request if needed
Task Management Workflow
Supervisors create and assign tasks; employees execute and update status through to completion.
1
Create Task Supervisor
  • Enters: Title, Description, Due Date (date picker), Priority (High / Medium / Low)
  • Selects one or multiple employees to assign the task
  • Task created with initial status: Pending
2
Employee Receives Task Employee
  • Task visible in employee's task list with title, description, due date, priority
  • Employee reviews requirements and begins work
  • Updates status to In Progress
3
Task Completion Employee
  • Work completed; employee updates status to Completed
  • Completion recorded with timestamp
4
Supervisor Oversight Supervisor
  • Monitors task status across the entire team
  • Can edit any task field (title, description, due date, priority)
  • Can delete a task with confirmation
  • Views team productivity rankings on the Leaderboard

1

2

Security & Authentication Layers
Multi-layered security covering token management, biometric controls, and location fraud prevention.
🔐
Token Storage
FlutterSecureStorage with hardware-level encryption protects JWT from unauthorized access
🎫
API Authentication
JWT Bearer token attached to every single API request — no unauthenticated calls permitted
📱
Device Binding
Mobile device ID stored with user account — prevents account sharing across devices
📸
Photo Verification
Mandatory camera capture on every check-in AND check-out creates a visual identity record
📍
GPS Geofencing
Employee GPS must fall within polygon boundary — prevents remote or fraudulent check-ins
⏱️
Session Timeout
Token expiry automatically redirects to login screen — no stale sessions permitted
🚪
Device Registration Gate
All new devices require admin approval before account activation — prevents unauthorized access
🔑
No Hardcoded Credentials
All credentials entered at runtime — nothing stored in source code or build artifacts
